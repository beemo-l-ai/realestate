# 아파트 실거래가 AI 검색 플로우 가이드 (Apartment Search Flow)

사용자가 프롬프트 기반으로 특정 아파트의 특정 평형대 실거래가(예: "래미안 84제곱 거래가 알려줘")를 질문했을 때, AI(LLM)가 문서를 정확히 필터링하고 환각(Hallucination) 없이 답변을 생성하기 위해 고안된 **5단계 검색 플로우**입니다.

---

## 5-Step Search Flow

### Step 1. 아파트 메타데이터 조회 (Retrieve Metadata)
사용자의 요청에 "아파트 이름"(예: 래미안)과 "지역"(예: 분당)이 포함되어 있다면, 가장 먼저 `search_apartment_metadata` MCP 도구를 호출합니다.
- **입력 (Input):** `districtName="분당구"`, `nameContains="래미안"`
- **출력 (Output):** 해당 구에 존재하는 모든 래미안 아파트 리스트, 각 아파트의 정확한 법정동명 밎 단지명(`apartmentName`), 그리고 존재하는 전용면적 배얄(`availableAreas: [59.9, 84.8, 114.2]`).

### Step 2. 메타데이터 필터링 및 추리기 (Filter Likely Matches)
LLM은 내부적으로 도구의 응답(메타데이터)을 분석하여 사용자가 찾고자 하는 아파트로 가장 유력한 후보군을 추립니다.
- "분당구"에 "래미안"이 여러 개 있다면 (예: 서현동 금호래미안, 정자동 래미안 등), 거래량이 가장 많거나 이름이 가장 유사한 후보를 선택합니다.
- 동시에 사용자가 요청한 "84제곱"이 해당 아파트의 `availableAreas` (예: `84.92`) 상에 존재하는지 매칭을 시도합니다.

### Step 3. 사용자 교차 검증 (User Confirmation)
정확한 아파트 단지를 특정하기 위해(Exact Match) LLM은 사용자에게 되묻습니다.
- **LLM 질문 예시:** *"분당구 정자동에 위치한 '래미안' 아파트를 찾으시는 게 맞나요? 이 단지의 84㎡(전용면적 84.92㎡) 데이터를 조회할까요?"*
- (만약 검색어가 명확하여 후보가 딱 1개라면 이 단계를 생략하고 4단계로 즉시 넘어갈 수 있습니다.)
- ChatGPT Apps UI를 사용하는 경우, 후보 리스트 버튼 선택으로도 동일한 교차 검증을 수행할 수 있습니다. 이때 선택 텍스트는 `select_apartment_candidate` 도구로 전달됩니다.

### Step 4. 정확한 명칭으로 실거래 문서 조회 (Query Transactions)
사용자가 아파트 단지를 확인(Confirm)하면, 메타데이터에서 확보한 **완벽히 일치하는 아파트 이름(`apartmentName`)**과 **매칭된 전용면적(`areaM2`)** 파라미터를 사용하여 `get_latest_transaction_examples` 도구를 호출합니다.
- **입력 (Input):** `districtName="분당구"`, `apartmentName="정자래미안"`, `areaM2=84.92`
- **목적:** 더 이상 59㎡나 114㎡가 섞인 평균값이 아닌, 84㎡ 평형에만 해당하는 순수 거래 내역 배열을 가져옵니다.

### Step 5. 데이터 추출 및 답변 생성 (Extract & Answer)
조회된 정확한 실거래 매매 기록들을 바탕으로, 사용자가 원하는 인사이트를 추출하여 자연어로 답변을 생성합니다.
- "최근 3개월 내 84㎡의 최고가는 OOO억 원이었으며, 평균 거래선은 OOO억 수준입니다." 와 같이 통계를 내거나 나열하여 완벽하게 개인화된 답변을 제공합니다.
